# watchctl

TUI-приложение для мониторинга кратковременных пиков нагрузки CPU, которые сложно поймать вручную.

## Возможности

- **Live-мониторинг** — текущая загрузка CPU с визуальным индикатором + список top процессов
- **Детектирование пиков** — автоматическая фиксация моментов превышения порога (50% CPU)
- **Дедупликация** — одинаковые пики в пределах 30 секунд не дублируются
- **История** — все события сохраняются в JSON, загружаются при следующем запуске
- **Анализ** — повторяющиеся процессы, новые процессы в пиках, summary
- **Логирование** — JSON-файлы в temp-директории с ротацией (последние 50)

## Запуск

```bash
go run ./cmd/watchctl/
```

Или сборка и установка:

```bash
go build -o watchctl.exe ./cmd/watchctl/
go install ./cmd/watchctl/
```

## Экраны

### Live
Текущая загрузка CPU с цветным индикатором (зелёный → жёлтый → красный), таблица top-процессов, индикатор недавнего пика.

### History
Список зафиксированных пиков (время, CPU%, основной процесс) + панель summary с повторяющимися и новыми процессами.

### Details
Полный snapshot выбранного пика — все процессы с PID, именем и CPU%.

## Архитектура

```
collector (goroutine) → analyzer (goroutine) → logger (goroutine)
                      ↘                      ↘
                        TUI renderer (main thread)
```

Взаимодействие через каналы Go. UI не блокируется при сборе метрик.

## Логи

Файлы пиков сохраняются в `os.TempDir()/watchctl/`:

```json
{
  "id": "a1b2c3d4",
  "timestamp": "2026-04-16T14:30:00Z",
  "total_cpu": 78.5,
  "threshold": 50,
  "top_procs": [
    {
      "pid": 1234,
      "name": "chrome.exe",
      "cpu_percent": 45.2,
      "create_time": 1713264600000
    }
  ]
}
```
